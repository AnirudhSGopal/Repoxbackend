- [ ] `FRONTEND_URL` - Frontend public HTTPS domain (e.g., https://app.yourdomain.com)
- [ ] `ENVIRONMENT=production`
- [ ] `SECRET_KEY` - Strong random 64+ character hex
- [ ] `JWT_SECRET` - Strong random 64+ character hex
- [ ] `SESSION_SECRET` - Strong random 64+ character hex
- [ ] `GITHUB_CLIENT_ID` - From GitHub OAuth App
- [ ] `GITHUB_CLIENT_SECRET` - From GitHub OAuth App
- [ ] `GITHUB_APP_ID` - From GitHub App
- [ ] `GITHUB_WEBHOOK_SECRET` - From GitHub App

Optional but recommended:

## DEPLOYMENT STEPS

### Step 1: Prepare Coolify
```bash
# In Coolify dashboard:
1. Create PRGuard Backend service
2. Create PRGuard Frontend service
3. Configure Git webhook for auto-deploy
```

### Step 2: Configure Backend Service
```yaml
Service Type: Docker (Node.js build, Python runtime)
Repository: https://github.com/YourOrg/PRGuard
Branch: main
Root Directory: backend
Dockerfile: None (use default)
Start Command: python run.py
Port: 8000
```

### Step 3: Set Backend Environment Variables
```bash
# Copy all variables from PHASE 2 into Coolify environment
DATABASE_URL=postgresql+asyncpg://...
APP_URL=https://api.yourdomain.com
FRONTEND_URL=https://app.yourdomain.com
ENVIRONMENT=production
# ... etc
```

### Step 4: Configure Frontend Service
```yaml
Service Type: Node.js
Repository: https://github.com/YourOrg/PRGuard
Branch: main
Root Directory: frontend
Build Command: npm install && npm run build
Start Command: npm run preview
Port: 3000
```

### Step 5: Set Frontend Environment Variables
```bash
VITE_API_BASE_URL=https://api.yourdomain.com
VITE_GITHUB_CLIENT_ID=your_client_id
```

### Step 6: Configure Domains
```bash
# In Coolify:
Backend: api.yourdomain.com → Port 8000
Frontend: app.yourdomain.com → Port 3000
# Ensure HTTPS certificates auto-generated
```

### Step 7: Deploy
```bash
# Push to main branch
git push origin main

# Coolify auto-deploys both services
# Monitor logs for errors
```

### Step 8: Verify Deployment
```bash

# Check frontend loads

# Visit app, click login, redirect to GitHub, back to app
```

## TROUBLESHOOTING

| Issue | Solution |
|-------|----------|
| Database connection fails | Check DATABASE_URL format, test with psql, verify SSL |
| OAuth redirect fails | Verify APP_URL matches GitHub callback URL, check CORS |
| Frontend shows wrong API URL | Rebuild frontend with correct VITE_API_BASE_URL |
| Session not persisting | Check cookies in browser (should be httpOnly, Secure, SameSite) |
| Redis connection timeout | Either configure REDIS_URL or remove it (graceful degradation) |
| Admin login not working | Check ADMIN_USERNAME/ADMIN_PASSWORD set, verify admin user exists |
| CORS errors | Add FRONTEND_URL to CORS_ORIGINS, restart backend |
| /health returns 503 | Check database connection, redis not required |
